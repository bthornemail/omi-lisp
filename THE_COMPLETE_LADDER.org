# CHAPTER 1: THE SEED

## THE BEGINNING

We start with nothing. No variables. No definitions. No functions. No data structures.

We start with the axis: NULL.

NOT zero. Zero is a number. NULL is the position from which all positions are measured.

NOT empty. Empty is an absence. NULL is a presence - the axis itself.

## WHY PAIR?

A single point has no structure. A single point cannot move. A single point cannot compute.

To compute, we need DUALITY. We need TWO. Movement from one to the other.

So we pair NULL with itself.

But wait - pairing NULL with NULL gives us:

```
(NULL . NULL)
```

This is two NULLs. But are they the same? Are they different?

If both are NULL, there is no distinction. The pair is meaningless.

We need the first DISTINCTION. We need a symbol that is NOT NULL.

That's SOH - Start of Header.

## THE SEED IS THE PAIR

The seed is NOT (NULL . NULL).

The seed is (NUL . SOH).

NUL = NULL = the axis (0x00)
SOH = Start of Header = first distinction from NULL (0x01)

```
(NUL . SOH)
```

Two DIFFERENT symbols. The first binary distinction.

This pair is the foundation. Everything else unfolds from this pair.

# CHAPTER 2: THE EVENT SELECTOR

## THE DELTA LAW

From the seed (NUL . SOH), we apply the delta law repeatedly.

```
delta(x) = rotl(x,1) XOR rotl(x,3) XOR rotr(x,2) XOR C
```

Where:
- rotl(x,n) = rotate left by n bits
- rotr(x,n) = rotate right by n bits
- XOR = exclusive or
- C = constant = STX = 0x02

This operation is REVERSIBLE. No information is lost.

## THE 4 SYMBOLS

Each application of delta produces a new symbol.

After enough applications, we earn these 4 symbols:

```
NUL SOH STX ETX
```

- NUL (0x00) = NULL = the axis
- SOH (0x01) = Start of Header
- STX (0x02) = Start of Text
- ETX (0x03) = End of Text

## THE 32-BIT EVENT SELECTOR

These 4 symbols become the EVENT SELECTOR.

QEMU ACPI GED has a 32-bit event selector (ESEL) at memory address [0x0-0x3].

This is 4 bytes = 32 bits.

The 4 symbols (NUL, SOH, STX ETX) are the QUASI-GENERATOR of the 32-bit selector.

BIT 0 = NUL (memory hotplug event)
BIT 1 = SOH (power down event)
BIT 2 = STX (NVDIMM hotplug event)
BIT 3 = ETX (CPU hotplug event)
BITS 4-31 = remaining control codes

The 4 symbols generate the full 32-bit selector.

This is the first doubling: 4 → 32.

# CHAPTER 3: CONTROL / SIGNAL

## THE SECOND DOUBLING

From 4 symbols, we double to 8.

```
CONTROL (first 4): NUL SOH STX ETX
SIGNAL (next 4):   EOT ENQ ACK BEL
```

- EOT (0x04) = End of Transmission
- ENQ (0x05) = Enquiry
- ACK (0x06) = Acknowledge
- BEL (0x07) = Bell

## CONTROL = INITIATING

CONTROL = the initiating events.
Events that START something:
- NUL = axis
- SOH = start of header
- STX = start of text
- ETX = end of text

## SIGNAL = ACKNOWLEDGING

SIGNAL = the acknowledging events.
Events that RESPOND:
- EOT = end of transmission (response to start)
- ENQ = enquiry (request for response)
- ACK = acknowledge (positive response)
- BEL = bell (attention signal)

## PAIRING

```
(CONTROL . SIGNAL)
```

Pairs of pairs. The first layer of structure.

This is the second doubling: 4 → 8.

# CHAPTER 4: REFERENCES / POINTERS

## THE THIRD AND FOURTH DOUBLINGS

From 8 symbols, we double to 16.
From 16 symbols, we double to 32.

REFERENCES (first 16, 0x00-0x0F):
```
NUL SOH STX ETX EOT ENQ ACK BEL BS HT LF VT FF CR SO SI
```

- NUL (0x00) = NULL = the axis
- SOH (0x01) = Start of Header
- STX (0x02) = Start of Text
- ETX (0x03) = End of Text
- EOT (0x04) = End of Transmission
- ENQ (0x05) = Enquiry
- ACK (0x06) = Acknowledge
- BEL (0x07) = Bell
- BS (0x08) = Backspace
- HT (0x09) = Horizontal Tab
- LF (0x0A) = Line Feed
- VT (0x0B) = Vertical Tab
- FF (0x0C) = Form Feed
- CR (0x0D) = Carriage Return
- SO (0x0E) = Shift Out
- SI (0x0F) = Shift In

POINTERS (second 16, 0x10-0x1F):
```
DLE DC1 DC2 DC3 DC4 NAK SYN ETB CAN EM SUB ESC FS GS RS US
```

- DLE (0x10) = Data Link Escape
- DC1 (0x11) = Device Control 1
- DC2 (0x12) = Device Control 2
- DC3 (0x13) = Device Control 3
- DC4 (0x14) = Device Control 4
- NAK (0x15) = Negative Acknowledge
- SYN (0x16) = Synchronous Idle
- ETB (0x17) = End of Transmission Block
- CAN (0x18) = Cancel
- EM (0x19) = End of Medium
- SUB (0x1A) = Substitute
- ESC (0x1B) = Escape (the turn, the mode shift)
- FS (0x1C) = File Separator (Function Set)
- GS (0x1D) = Group Separator (Group Set)
- RS (0x1E) = Record Separator (Rule Set)
- US (0x1F) = Unit Separator (Unit Set)

## CONTROL PLANE = LUT

Together, REFERENCES and POINTERS form the FULL CONTROL PLANE.

32 control codes = 32 events = THE LUT.

These are NOT characters. These are NOT text.

These are runtime-dependent control signals. They are WHAT HAPPENS at runtime.

The 32 control codes ARE the LUT. The LUT IS the 32 control codes.

This is the third and fourth doubling: 8 → 16 → 32.

# CHAPTER 5: THE RUPTURE

## SP = SPACE

SP (0x20) is the 33rd symbol.

SP = SPACE = 0x20 = 32 decimal.

This is THE RUPTURE.

## WHY RUPTURE?

The control plane is 0x00-0x1F (32 symbols).

SP is 0x20 (33rd symbol).

When we reach SP, we TRANSPOSE. We LEAVE the control plane.

SP is the first TRANSPOSITION point.

## WHAT IS SP?

SP is just SP. Just a space character. The separator. The pause.

NOT ( ) . yet.

Those come LATER, at their positions in the stream.

SP creates the first empty position. The first pause in the stream.

## BEFORE AND AFTER

BEFORE SP: raw control signals (0x00-0x1F). No structure.

AFTER SP: structure becomes possible (0x20-0x7F).

SP is the rupture point. The transition.

## LISPGRAMMAR BECOMES POSSIBLE

At position 0x28 (8 steps AFTER SP), we earn ( - left paren.

Now we can write:
- (a b) - lists
- (a . b) - pairs
- 'a - quotes
- ; - comments
- : - labels

This is when the kernel initializes. When Lisp becomes possible.

# CHAPTER 8: ASCII

## FULL ASCII

ASCII = 128 characters = 0x00 to 0x7F.

- Control codes: 0x00-0x1F (32 symbols)
- Non-printing: 0x20-0x3F (32 symbols)
- Printing: 0x40-0x7F (64 symbols)

Total: 128 symbols.

This is the ASCII block. This is when the system is fully booted.

# CHAPTER 10: BRAILLE

## THE BRAILLE BLOCK

Braille is U+2800 to U+28FF.

This is 256 codepoints.

Braille patterns:
- Dot 1 = 0x01
- Dot 2 = 0x02
- Dot 3 = 0x04
- Dot 4 = 0x08
- Dot 5 = 0x10
- Dot 6 = 0x20
- Dot 7 = 0x40
- Dot 8 = 0x80

Each Braille character is a pattern of 1-8 dots.

Braille is ONE representation of the symbol space.

# CHAPTER 11: AEGEAN

## THE AEGEAN BLOCK

Aegean is U+10100 to U+1013F.

This is 64 codepoints.

Aegean numerals are bijective base-9. No zero.

```
U+10100 = 1
U+10101 = 2
...
U+1013F = 64
```
